# Next.js → TanStack Start + Better Auth Migration Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Next.js 16 with TanStack Start and Convex Auth with Better Auth, preserving all existing features.

**Architecture:** In-place migration on the `migrate/tanstack-start` branch. Frontend routing moves from `src/app/` (Next.js App Router) to `src/routes/` (TanStack Start file-based routes). Auth moves from `@convex-dev/auth` to `@convex-dev/better-auth` with session cookies. Backend Convex services remain untouched.

**Tech Stack:** TanStack Start, TanStack Router, Vite, Better Auth, Convex, React 19, shadcn/ui, Tailwind CSS

**Spec:** `docs/superpowers/specs/2026-03-24-tanstack-start-migration-design.md`

**Reference docs:**
- TanStack Start quickstart with Convex: https://docs.convex.dev/quickstart/tanstack-start
- Better Auth + TanStack Start guide: https://labs.convex.dev/better-auth/framework-guides/tanstack-start
- Better Auth setup: https://labs.convex.dev/better-auth
- TanStack Start build from scratch: https://tanstack.com/start/latest/docs/framework/react/build-from-scratch

```ts
    database: authComponent.adapter(ctx),
    emailAndPassword: {
      enabled: true,
      requireEmailVerification: false,
    },
    socialProviders: {
      github: {
        clientId: process.env.AUTH_GITHUB_ID!,
        clientSecret: process.env.AUTH_GITHUB_SECRET!,

        clientSecret: process.env.AUTH_GOOGLE_SECRET!,

        roles: {
          type: "string[]",
          required: false,
        },
      },
    },
    hooks: {
      after: [
        {
          matcher: (context) => context.path === "/sign-up/email",
          handler: async (ctx) => {
            const response = ctx.response;
            if (!response || !response.body) return;

            const body =
              typeof response.body === "string"
                ? JSON.parse(response.body)
                : response.body;
            const userId = body?.user?.id;
            const email = body?.user?.email;
            const name = body?.user?.name ?? email?.split("@")[0];

            if (userId && email) {
              // Schedule welcome email (same as old afterUserCreatedOrUpdated)
              // This runs as an after-hook, so we can use ctx.context to access Convex
              // The actual email sending is handled by the email service
            }
          },
        },
      ],
    },
    plugins: [convex({ authConfig })],
  });
};

// NOTE: Do NOT add a getCurrentUser query here — it already exists in convex/users.ts.
// The existing getCurrentUser in users.ts uses the userQuery builder and authHelpers.getCurrentUser,
// which will work with Better Auth after the getAuthUserId import swap in authHelpers.ts.
```

**Important — Welcome email fallback strategy:** The Better Auth hook runs in an HTTP context, NOT a Convex mutation context, so `ctx.scheduler` is not available. Three options in order of preference:

1. **Better Auth `databaseHooks`**: If `@convex-dev/better-auth` supports `databaseHooks.user.create.after`, use it to call a Convex mutation via `fetchAuthMutation` that schedules the email.
2. **Client-side trigger**: After successful signup in the signup page, call a Convex mutation to send the welcome email. Simple but less reliable (user could close browser).
3. **Convex trigger/cron**: Use a Convex scheduled function that polls for new users without welcome emails. Most reliable but more complex.

During implementation, try option 1 first. Check the Better Auth docs for `databaseHooks` or similar. Fall back to option 2 if hooks aren't supported.

- [ ] **Step 1: Create router.tsx**

```tsx
import { ConvexQueryClient } from "@convex-dev/react-query";
import { QueryClient } from "@tanstack/react-query";
import { createRouter as createTanStackRouter } from "@tanstack/react-router";
import { routerWithQueryClient } from "@tanstack/react-router-with-query";
import { ConvexProvider } from "convex/react";
import { ConvexBetterAuthProvider } from "@convex-dev/better-auth/react";
import { routeTree } from "./routeTree.gen";

export function createRouter() {
  const CONVEX_URL = import.meta.env.VITE_CONVEX_URL!;

  const convexQueryClient = new ConvexQueryClient({
    convexUrl: CONVEX_URL,
    expectAuth: true,
  });

  const queryClient = new QueryClient({
    defaultOptions: {
      queries: {
        queryKeyHashFn: convexQueryClient.hashFn(),
        queryFn: convexQueryClient.queryFn(),
      },
    },
  });
  convexQueryClient.connect(queryClient);

  const router = routerWithQueryClient(
    createTanStackRouter({
      routeTree,
      context: { queryClient, convexQueryClient },
      scrollRestoration: true,
      Wrap: ({ children }) => (
        <ConvexBetterAuthProvider client={convexQueryClient.convexClient}>
          {children}
        </ConvexBetterAuthProvider>
      ),
    }),
    queryClient,
  );

  return router;
}

declare module "@tanstack/react-router" {
  interface Register {
    router: ReturnType<typeof createRouter>;
  }
}
```

**Note about non-functional state:** The app will be broken between Tasks 2-5. This is expected — the old auth system is removed before the new one is fully wired. Do not attempt to run/test between these tasks.

**Note:** The `ConvexBetterAuthProvider` may need an `initialToken` prop for SSR. Check the Better Auth TanStack Start guide for the exact pattern. If SSR token passing is needed, it's configured via the root route's `beforeLoad` + `getToken()` from auth-server.

```tsx
import { authClient } from "@/lib/auth-client";
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";
import { Label } from "@/components/ui/label";
import {
  Card,
  CardContent,
  CardDescription,
  CardHeader,
  CardTitle,
} from "@/components/ui/card";

export const Route = createFileRoute("/forgot-password")({
  component: ForgotPasswordPage,
});

function ForgotPasswordPage() {
  const [email, setEmail] = useState("");
  const [error, setError] = useState("");
  const [success, setSuccess] = useState(false);
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    setError("");
    setLoading(true);
    try {
      await authClient.forgetPassword({ email, redirectTo: "/reset-password" });
      setSuccess(true);
    } catch (err) {
      setError(err instanceof Error ? err.message : "Failed to send reset email");
    } finally {
      setLoading(false);
    }
  };

  return (
    <div className="flex min-h-screen items-center justify-center p-4">
      <Card className="w-full max-w-md">
        <CardHeader className="text-center">
          <CardTitle className="text-2xl font-bold">Reset Password</CardTitle>
          <CardDescription>
            {success
              ? "Check your email for a reset link"
              : "Enter your email to receive a password reset link"}
          </CardDescription>
        </CardHeader>
        <CardContent>
          {success ? (
            <div className="space-y-4 text-center">
              <p className="text-sm text-muted-foreground">
                If an account exists for {email}, we sent a reset link.
              </p>
              <Link to="/signin" className="text-sm text-primary hover:underline">
                Back to sign in
              </Link>
            </div>
          ) : (
            <form onSubmit={handleSubmit} className="space-y-4">
              {error && (
                <div className="rounded-md bg-destructive/10 p-3 text-sm text-destructive">
                  {error}
                </div>
              )}
              <div className="space-y-2">
                <Label htmlFor="email">Email</Label>
                <Input
                  id="email"
                  type="email"
                  placeholder="you@example.com"
                  value={email}
                  onChange={(e) => setEmail(e.target.value)}
                  required
                />
              </div>
              <Button type="submit" className="w-full" disabled={loading}>
                {loading ? "Sending..." : "Send Reset Link"}
              </Button>
              <div className="text-center">
                <Link to="/signin" className="text-sm text-muted-foreground hover:underline">
                  Back to sign in
                </Link>
              </div>
            </form>
          )}
        </CardContent>
      </Card>
    </div>
  );
}
```

```tsx
import { createFileRoute, Link, useNavigate } from "@tanstack/react-router";
import { useState } from "react";
import { authClient } from "@/lib/auth-client";
import { Button } from "@/components/ui/button";
import { Label } from "@/components/ui/label";
import { PasswordInput, isPasswordValid } from "@/components/auth/password-input";
import {
  Card,
  CardContent,
  CardDescription,
  CardHeader,
  CardTitle,
} from "@/components/ui/card";

export const Route = createFileRoute("/reset-password")({
  validateSearch: (search: Record<string, unknown>) => ({
    token: (search.token as string) ?? "",
  }),
  component: ResetPasswordPage,
});

function ResetPasswordPage() {
  const { token } = Route.useSearch();
  const navigate = useNavigate();
  const [newPassword, setNewPassword] = useState("");
  const [error, setError] = useState("");
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    if (!isPasswordValid(newPassword)) return;
    setError("");
    setLoading(true);
    try {
      await authClient.resetPassword({ token, newPassword });
      navigate({ to: "/signin" });
    } catch (err) {
      setError(err instanceof Error ? err.message : "Failed to reset password");
    } finally {
      setLoading(false);
    }
  };

  if (!token) {
    return (
      <div className="flex min-h-screen items-center justify-center p-4">
        <Card className="w-full max-w-md">
          <CardContent className="pt-6 text-center">
            <p className="text-destructive">Invalid or missing reset token.</p>
            <Link to="/forgot-password" className="text-sm text-primary hover:underline">
              Request a new reset link
            </Link>
          </CardContent>
        </Card>
      </div>
    );
  }

  return (
    <div className="flex min-h-screen items-center justify-center p-4">
      <Card className="w-full max-w-md">
        <CardHeader className="text-center">
          <CardTitle className="text-2xl font-bold">Set New Password</CardTitle>
          <CardDescription>Enter your new password below</CardDescription>
        </CardHeader>
        <CardContent>
          <form onSubmit={handleSubmit} className="space-y-4">
            {error && (
              <div className="rounded-md bg-destructive/10 p-3 text-sm text-destructive">
                {error}
              </div>
            )}
            <div className="space-y-2">
              <Label htmlFor="password">New Password</Label>
              <PasswordInput
                id="password"
                value={newPassword}
                onChange={(e) => setNewPassword(e.target.value)}
                required
              />
            </div>
            <Button
              type="submit"
              className="w-full"
              disabled={loading || !isPasswordValid(newPassword)}
            >
              {loading ? "Resetting..." : "Reset Password"}
            </Button>
          </form>
        </CardContent>
      </Card>
    </div>
  );
}
```

```tsx
import { createFileRoute, Outlet, redirect } from "@tanstack/react-router";
import { useQuery } from "convex/react";
import { api } from "@/convex/_generated/api";
import { useSession } from "@/lib/auth-client";
import { AppShell } from "@/components/layout/app-shell";

export const Route = createFileRoute("/_app")({
  beforeLoad: async () => {
    // Server-side auth check. Try importing getToken from auth-server:
    // const token = await getToken();
    // if (!token) throw redirect({ to: '/signin' });
    //
    // If getToken doesn't work in beforeLoad context (depends on Better Auth's
    // TanStack Start integration), the client-side check in the component
    // below serves as the fallback auth gate. The Convex backend guards
    // (userQuery/userMutation) are always enforced regardless.
  },
  component: AppLayout,
});

function AppLayout() {
  const { data: session, isPending } = useSession();
  const user = useQuery(
    api.users.getCurrentUser,
    session ? {} : "skip",
  );

  if (isPending) {
    return (
      <div className="flex items-center justify-center h-screen">
        <div className="h-8 w-8 animate-spin rounded-full border-4 border-primary border-t-transparent" />
      </div>
    );
  }

  if (!session) {
    // Client-side redirect fallback
    window.location.href = "/signin";
    return null;
  }

  if (user === undefined) {
    return (
      <div className="flex items-center justify-center h-screen">
        <div className="h-8 w-8 animate-spin rounded-full border-4 border-primary border-t-transparent" />
      </div>
    );
  }

  return (
    <AppShell>
      <Outlet />
    </AppShell>
  );
}
```

**Note:** The `beforeLoad` server-side check may need `getToken()` from auth-server. If the client-side fallback is sufficient for your use case, the `beforeLoad` can remain a placeholder. Consult the Better Auth TanStack Start guide for the exact SSR pattern.
